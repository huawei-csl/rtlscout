# `rtl_rewriter` benchmarks

All fourteen RTLRewriter (ICCAD 2024, arXiv:2409.11414) short-bench cases, scaffolded in the
same shape as `benchmarks/turbo_rtl/<case>/`. The cost metric is `yosys_wires`
and/or `yosys_cells` (tech-independent post-`synth` counts — see
`core/cost.py`); correctness is guarded by a data-driven `tb.sv` +
`vectors.dat` captured from the baseline (see *Correctness semantics*
below).

Per-case confidence (`high`/`medium`/`low`) is recorded in
[`rewriter_cases.json`](rewriter_cases.json); the full rubric lives at
`meta.confidence_levels` there. `low` here means *the absolute Yosys
wire/cell numbers drift from the paper* (typically a Yosys-version
artefact), not that the directory attribution is uncertain — the
paper-position → upstream-directory mapping is unambiguous for all 14
cases.

`tb.sv` column is `PASS` when the baseline passes its own generated
testbench at 100% — 14/14 today.

The yosys script we use appends `clean -purge` after `synth` before calling
`stat`, so public alias buffers and dangling nets that yosys's default
`opt_clean` preserves for debuggability (e.g. the `sum_reg → sum` alias
that SpireHDL-style registered outputs leave behind) are dropped from the
count. This only affects `wires` — `cells` is unchanged either way — and
makes the numbers comparable across emitters that differ in how many
named intermediates they surface.

## Per-case layout

```
benchmarks/rtl_rewriter/<case_id>/
  description.txt                   # spec + reference numbers + cost metric
  metadata.json                     # name, module_name, confidence, reference, source, tb_mode
  tb.sv                             # self-checking data-driven testbench (reads vectors.dat)
  vectors.dat                       # 2000 random-stimulus vectors captured from the baseline
  context/
    starting_point.v                # verbatim baseline from RTLRewriter-Bench
```

`tb.sv` and `vectors.dat` are regenerated by `/tmp/rtl_rewriter_tb_gen.py`:
for each case, verilator compiles the baseline against a probe testbench,
drives 2000 LFSR-seeded random inputs (plus the all-zero and all-ones
corners), captures the baseline's outputs, and writes the data-driven
self-checking `tb.sv`. `metadata.json.tb_mode` is `comb` for the six
combinational cases and `seq` for the four clocked ones; the latter also
carry `clock_port` and (optionally) `reset_port`.

Note: case1's upstream file only explicitly declares `reg reg_a, reg_b,
reg_c` and relies on yosys to auto-infer `reg_d..reg_i` from their use
inside `always @(posedge clk)`. Verilator rejects this, so the shipped
`context/starting_point.v` adds an explicit `reg reg_d, reg_e, reg_f,
reg_g, reg_h, reg_i;` declaration — synthesis output is unchanged (28/18
wires/cells before and after).

Note: case5's upstream `adder_bit_width.v` declares `output [8:0] sum`
(implicit wire) and procedurally assigns to it inside `always @(a or b)`.
Verilator rejects this as PROCASSWIRE, so the shipped
`context/starting_point.v` changes the declaration to
`output reg [8:0] sum`; the Yosys synth output is unchanged (43 / 49
wires/cells before and after).

Note: for case8, case12, and case14 the upstream `*_ours.v` file is
0 bytes (and for case14 the filename is itself typo'd —
`mux_type_ours.v`, missing the "4"). In each of those cases the
metadata's `source.rtlr_file` is substituted with the human-golden
`*_optimized.v` variant from the same upstream directory; the
`source.rtlr_file_note` field records the substitution and any quirks
(case8's `optimized_multiplier` differs in top-module name from the
baseline `inefficient_multiplier`; case14 uses
`mux_type4_redundancy_optimzied.v`, which is byte-identical to its
sibling `_improve.v` and carries an upstream `optimzied` typo).

Note: case11's upstream `mux_dead_code.v` declares `sum_resullt` (typo)
but instantiates the adder onto `sum_result`. Both Verilator (with
`-Wno-fatal`) and Yosys auto-create the implicit `sum_result` net — the
functional behavior and synthesized netlist are unchanged, so the file
ships unmodified. The paper's case11 numbers (34/35) come from a
flattened synth under Yosys 0.33; our framework pipeline keeps the
`and_bitwise`/`or_bitwise` submodules separate and totals 13/24. The
adder/subtractor/alu submodules are pruned as dead code in both flows.
Upstream `mux_dead_code_ours.v` is 0 bytes, so the rtlr-side comparison
cannot be reproduced from the public repo.

`metadata.json.source` points back at the upstream file so the scaffold can be
regenerated deterministically:

```json
{
  "source": {
    "repo":     "https://github.com/yaoxufeng/RTLRewriter-Bench",
    "commit":   "96639fe67bc8f020c05978ef5028d75fcf6980f2",
    "path":     "short_benchmark/basic/add3/add3.v",
    "rtlr_file":"short_benchmark/basic/add3/add3_ours.v"
  }
}
```

## Running one

Full eval (correctness + cost) via the framework's `run_eval.py`:

```bash
~/pyenv_eda/bin/python run_eval.py \
    benchmarks/rtl_rewriter/case1/context/starting_point.v \
    --benchmark benchmarks/rtl_rewriter/case1 \
    --language verilog --cost-metric yosys_cells
```

Expected: `Correctness: PASS, 2000/2000` (or 2002/2002 for the six
combinational cases, which include the all-zero and all-ones corners) and
the `yosys_cells` / `yosys_wires` value recorded in
`metadata.json.reference`. Note the turbo_rtl in-place artifact warning
applies here too — prefer `--workdir /tmp/foo` over running in `context/`.

Cost-metric-only (skip correctness):

```bash
python -c "
from pathlib import Path
import json
from core.cost import make_cost_metric

case = 'case1'
d   = Path(f'benchmarks/rtl_rewriter/{case}')
md  = json.loads((d/'metadata.json').read_text())
sp  = d/'context/starting_point.v'

wires = make_cost_metric('yosys_wires').evaluate(d, top_module=md['module_name'], design_file=sp)
cells = make_cost_metric('yosys_cells').evaluate(d, top_module=md['module_name'], design_file=sp)
print(case, 'wires=', int(wires.value), 'cells=', int(cells.value), 'stats=', wires.stats)
"
```

Both metrics share a single `yosys -p '…; synth; stat'` backend and expose
both `wires` and `cells` in `CostResult.stats` regardless of which is the
cost value, so you only need one call to get both numbers in production.

## Correctness semantics

Random-stimulus `tb.sv` is the first line of defence — fast, catches
anything that's not bit-identical to the baseline on 2000 cycles of
coverage. For a stronger guarantee (total equivalence, not just sample
coverage), layer a yosys `equiv_make; equiv_induct` check on top:

```bash
yosys -p "
  read_verilog -sv benchmarks/rtl_rewriter/case1/context/starting_point.v
  rename example gold
  read_verilog -sv path/to/rewrite.v
  rename example gate
  equiv_make gold gate equiv
  hierarchy -top equiv
  equiv_simple
  equiv_status -assert
"
```

Coverage: the FSMs (case1/case9/case10/case11) are driven with random
inputs after a 3-cycle reset, so reset-sensitive behaviour at cycle 0 is
not covered by `tb.sv` — equiv-check if that matters.
